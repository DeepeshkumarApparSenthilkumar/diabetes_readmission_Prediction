# Diabetes 30-Day Readmission Prediction

Predicting 30-day hospital readmission in diabetic patients using 10 years of
clinical encounter data from 130 US hospitals (1999–2008). Compares Logistic
Regression, Random Forest, XGBoost, LightGBM, and a stacking ensemble with
threshold optimization.

**Dataset:** [UCI Diabetes 130-US Hospitals](https://archive.ics.uci.edu/dataset/296/diabetes+130-us+hospitals+for+years+1999-2008) — 101,766 encounters, 50 features

## Key Findings

- **Discharge disposition** is the strongest predictor — where a patient goes after
  discharge matters more than most in-hospital features (top SHAP feature with
  mean |SHAP| = 0.246)
- **Prior inpatient visits** (≥3) identify a high-risk subgroup with 2–3× average
  readmission rate (mean |SHAP| = 0.133)
- **Removing deceased/hospice patients** fixes a data leakage issue that inflated
  the negative class — shifts positive rate from 8.8% → 7.1%
- **SMOTE** applied to training set only (K=5) to address 13:1 class imbalance
  without contaminating the test evaluation

## How to Run

### Option A — one-shot (recommended)

```r
setwd("path/to/diabetes_readmission_Prediction")
source("run_pipeline.R")   # runs all 7 R scripts, ~20-30 min
```

Then run Python SHAP and knit the report:


```r
rmarkdown::render("docs/report.Rmd", output_file = "report.html")
```

### Option B — step by step

```r
setwd("path/to/diabetes_readmission_Prediction")
source("R/01_data_audit.R")            # ~30 s   — audit + missing_audit.csv
source("R/02_preprocessing.R")         # ~20 s   — diabetes_clean.csv
```

## Pipeline Overview

```
diabetic_data.csv (101,766 rows)
        │
        ▼ 01_data_audit   → missing_audit.csv, cardinality.csv
        │
        ▼ 02_preprocessing → diabetes_clean.csv
          • remove deceased/hospice (discharge IDs 11,13,14,19,20,21)
          • deduplicate: first encounter per patient → 71,518 rows
          • ICD-9 → 9 diagnosis groups; age → numeric midpoint
          • binary target: readmitted_binary = (readmitted == "<30")
          • positive rate: 7.1% (13:1 imbalance)
        │
        ▼ 03_eda          → figures 01-16, statistical_tests.csv
        │
        ▼ 04_feature_engineering → diabetes_featured.csv (+8 features)
          high_utilizer, polypharmacy, total_visits, diab_primary,
          any_change, age_x_inpatient, med_x_diagnoses, emergency_ratio
        │
        ▼ 05_modeling (70/30 split + SMOTE K=5 on train only)
          LR (glmnet, lambda grid) | RF (mtry grid, ntree=200)
          XGBoost (max_depth×eta grid, 5-fold CV + early stopping)
          LightGBM (num_leaves×lr grid, 5-fold CV + early stopping)
        │
        ▼ 06_ensemble → stacking GLM on OOF probabilities
          + threshold optimization (F1-optimal & recall≥70%)
        │
        ▼ 07_evaluation → model_comparison.csv, figures 17-21
          AUC · PR-AUC · Precision · Recall · Specificity · NPV · F1
          DeLong test: Ensemble vs each base model
        │
        ▼ python/shap_analysis.py → figures 22-24, shap_values.csv
          SHAP TreeExplainer on XGBoost JSON model
```
